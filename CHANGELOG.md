# Changelog

## [0.1.0] — 2026-04-14

### Added

- **Sidebar panel** — health score card, active branch info, stale branch manager (merge / delete / revive), and AI-powered suggestions, all in a single VS Code sidebar view.
- **Local-first inference** — connects to Ollama, LM Studio, Jan.ai, or any llama.cpp-compatible server running on localhost; no data leaves your machine.
- **Collect-all provider probing** — discovers every running local LLM provider in parallel and lets you pick the provider and model via a two-step QuickPick (`BranchMind: Select Local Model`).
- **Rule-based fallback** — when no local model is running, branch naming and convention suggestions are still generated by a deterministic rules engine (zero latency, zero tokens).
- **Secret redaction layer** — 14 regex patterns (AWS, OpenAI, GitHub PAT, Slack, Firebase, Supabase, Stripe, Razorpay, generic secrets, connection strings, `.env` pairs, non-localhost IPs) strip credentials before any diff reaches inference.
- **Branch health audit** — scores your repo 0–100 by counting stale branches, naming inconsistencies, and untracked remotes; cached to `.branchmind/audit.json` and refreshed only when new commits appear.
- **fs.watch branch detection** — switches are detected instantly via `fs.watch('.git/HEAD')` (no polling); offers to stash uncommitted changes if you switch away with a dirty working tree.
- **Monorepo support** — detects nx / Turborepo / Lerna / pnpm-workspace layouts and scopes analysis to the active editor's package root.
- **Suggestion decay** — suggestions are marked stale with an amber badge after 5 minutes of no git activity, prompting a re-check.
- **Offline / degraded mode** — caches the last 5 suggestions so the panel stays useful when your local model goes offline.
- **`BranchMind: Re-scan`** command — re-probes all providers and refreshes the sidebar on demand.
